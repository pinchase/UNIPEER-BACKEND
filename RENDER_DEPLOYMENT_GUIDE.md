## 🐛 Fix for Build Error

**Error:** `Unknown server host 'unipeer-db-kagiripeterson8404-unipeer.j.aivencloud.com'`

**Root Cause:** Django was trying to connect to the database during the build phase, but:
1. The database hostname couldn't be resolved during build
2. The build environment doesn't have network access to external databases
3. Migrations were running before the database was available

**Solution Implemented:** ✅
- ✅ Created `build.sh` - Runs during build (no DB connection needed)
- ✅ Created `start.sh` - Runs on startup (after DB is available)
- ✅ Fixed `settings.py` - Falls back to SQLite if DATABASE_URL not set
- ✅ Fixed SSL check - Only checks HOST if it exists (not for SQLite)

## 📋 Quick Fix Steps

### Option A: Use Render PostgreSQL (Recommended) ✅

This is the **easiest and most reliable** option:

1. **In Render Dashboard:**
   - Go to your backend service
   - Click "Environment" tab
   - **REMOVE** the old `DB_URL` variable (Aiven database)
   - Add new variable: `DATABASE_URL` (leave it empty for now)

2. **Create a PostgreSQL Database:**
   - In Render dashboard, click "New +" → "PostgreSQL"
   - Name: `unipeer-db`
   - Plan: Free (or paid)
   - Click "Create Database"

3. **Link Database to Service:**
   - Go back to your backend service
   - Click "Environment" tab
   - Edit the `DATABASE_URL` variable
   - Select "Link to Database" → Choose `unipeer-db`
   - This will automatically populate the connection string

4. **Update Build Settings:**
   - Go to "Settings" tab
   - Build Command: `./build.sh`
   - Start Command: `./start.sh`
   - Click "Save Changes"

5. **Deploy:**
   - Click "Manual Deploy" → "Deploy latest commit"
   - Or push a new commit to trigger auto-deploy

### Option B: Keep Using Aiven MySQL

If you want to keep your Aiven database:

1. **In Render Dashboard:**
   - Go to your backend service
   - Click "Environment" tab
   - **RENAME** `DB_URL` to `DATABASE_URL`
   - Make sure the value is the full connection string:
     ```
     mysql://user:password@host:port/database
     ```

2. **Update Build Settings:**
   - Settings → Build Command: `./build.sh`
   - Settings → Start Command: `./start.sh`
   - Save Changes

3. **Verify ca.pem exists:**
   - Make sure `ca.pem` file is in your repository root
   - This is needed for SSL connection to Aiven

4. **Deploy:**
   - Push changes and redeploy

### Option C: Use SQLite (Development Only)

For testing only, not recommended for production:

1. **In Render Dashboard:**
   - Remove the `DB_URL` or `DATABASE_URL` variable
   - This will use SQLite (file-based database)

2. **Update Build/Start:**
   - Build Command: `./build.sh`
   - Start Command: `./start.sh`

⚠️ **Warning:** SQLite on Render will **lose data on redeploys**!

## 🔧 Environment Variables Setup

Here's the complete list of environment variables you need:

### Required Variables:

```bash
# Database (choose one option)
DATABASE_URL=postgresql://user:pass@host:5432/dbname  # Render PostgreSQL
# OR
DATABASE_URL=mysql://user:pass@host:3306/dbname       # Aiven MySQL

# Security
SECRET_KEY=<auto-generated-by-render>  # Use "Generate Value" button
DEBUG=False

# Allowed hosts
ALLOWED_HOSTS=.onrender.com,your-custom-domain.com
```

### Optional Variables:

```bash
# CORS for additional domains
ADDITIONAL_CORS_ORIGINS=https://preview-abc.vercel.app,https://preview-xyz.vercel.app

# Custom Python version (if needed)
PYTHON_VERSION=3.12.0
```

## 📝 Build and Start Commands

### Build Command (runs during deployment):
```bash
./build.sh
```

**What it does:**
- Installs Python dependencies
- Collects static files (using SQLite, no DB needed)
- Doesn't run migrations (DB not available yet)

### Start Command (runs when service starts):
```bash
./start.sh
```

**What it does:**
- Runs database migrations (DB is now available)
- Creates superuser if needed
- Starts Gunicorn web server

## 🐛 Troubleshooting

### Error: "Unknown server host"
**Cause:** Trying to connect to database during build  
**Fix:** Use `build.sh` as build command (it doesn't connect to DB)

### Error: "mysqlclient installation failed"
**Cause:** Missing system dependencies for MySQL  
**Fix:** Switch to PostgreSQL (recommended) or add to `render.yaml`:
```yaml
buildCommand: |
  apt-get update && apt-get install -y default-libmysqlclient-dev
  ./build.sh
```

### Error: "No module named 'MySQLdb'"
**Cause:** `mysqlclient` not installed  
**Fix:** It's in `requirements.txt`, should auto-install. If not, rebuild.

### Error: "table does not exist"
**Cause:** Migrations didn't run  
**Fix:** Check logs to ensure `start.sh` ran migrations successfully

### Build succeeds but app crashes on startup
**Cause:** Database connection issue  
**Fix:** 
1. Verify `DATABASE_URL` is set correctly
2. Check logs for connection errors
3. For Aiven: Ensure `ca.pem` exists and SSL settings are correct

### Static files not loading
**Cause:** Whitenoise not configured or collectstatic failed  
**Fix:** `build.sh` should handle this. Check build logs.

## 📊 Deployment Checklist

Before deploying:

- [ ] `build.sh` exists and is executable (`chmod +x build.sh`)
- [ ] `start.sh` exists and is executable (`chmod +x start.sh`)
- [ ] `settings.py` uses `DATABASE_URL` (not `DB_URL`)
- [ ] `requirements.txt` includes all dependencies
- [ ] Environment variables set in Render
- [ ] Database created and linked
- [ ] Build Command set to: `./build.sh`
- [ ] Start Command set to: `./start.sh`
- [ ] `SECRET_KEY` is generated (not hardcoded)
- [ ] `DEBUG=False` in production
- [ ] `ALLOWED_HOSTS` includes Render domain

After deploying:

- [ ] Service shows "Live" status
- [ ] Logs show successful migration
- [ ] Public endpoints return data
- [ ] Authentication is required
- [ ] Frontend can connect to backend
- [ ] No CORS errors in browser console

## 🆘 Still Having Issues?

### Check Logs:
1. In Render dashboard, go to your service
2. Click "Logs" tab
3. Look for error messages

### Common Log Messages:

**"Collecting static files..."** ✅ Good - Build phase working

**"Running database migrations..."** ✅ Good - Start phase working

**"OperationalError: could not connect"** ❌ Bad - Database connection issue
- Check `DATABASE_URL` is set
- Verify database is running
- Check firewall/network settings

**"ModuleNotFoundError"** ❌ Bad - Dependency missing
- Check `requirements.txt`
- Rebuild from scratch

**"ALLOWED_HOSTS"** ❌ Bad - Domain not allowed
- Add your domain to `ALLOWED_HOSTS` env var
